Sobre Git y Github - 27/1172025
Git es un sistema para controlar versiones de un proyecto de desarrollo web. Es práctico porque permite el trabajo colaborativo, guardar distintas versiones de un proyecto y tener un respaldo ante cualquier eventualidad. Por su parte, GitHub es el lugar en la web donde se aloja todo lo respaldado en Git. Permite dar acceso público a los proyectos e incluso clonarlos para trabajar en ellos. Así como otros profesionales tienen sus portafolios de trabajos realizados, un desarrollador cuenta con GitHub para documentar sus proyectos de software.

Sobre los comandos:
git —version / Última versión instalada de Git en el sistema.
git —config / Aquí agregas tus credenciales (usuario y correo) para “vincular” Git con tu GitHub.
git —init / Sirve para iniciar un repositorio.
git —add / Agregar archivos.
git —commit / Sirve para realizar cambios y muestra si no los hemos subido (cada cambio en el código se va mostrando).
git —status / Muestra el estado actual del repositorio.
